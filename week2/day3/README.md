# 🚀 MEDALLION ARCHITECTURE – PYSPARK DATA PIPELINE

# 📌 OVERVIEW

This project demonstrates a complete data engineering workflow using the Medallion Architecture approach in PySpark and Databricks.

The pipeline processes sales transaction data through:
- Data Cleaning
- Data Validation
- Aggregations
- Customer Insights
- Data Quality Checks
- Latest Record Handling
- Dashboard-Ready Gold Tables

The final outputs are stored as Delta Tables for reporting and analytics tools like Power BI.

---

# 🛠️ TECHNOLOGIES USED

- PySpark
- Databricks Community Edition
- Delta Lake
- DataFrames
- Window Functions
- Aggregations

---

# 📂 DATASET USED

The dataset contains sales transaction records with the following fields:

- order_id
- customer_id
- product
- category
- city
- date
- amount
- quantity

# ⚙️ PIPELINE OBJECTIVES

The project focuses on:
- Cleaning invalid data

- Performing business aggregations

- Creating dashboard-ready datasets

# 📚 REQUIREMENTS IMPLEMENTED

# 🔹 REQUIREMENT 1 – SALES ANALYSIS

## 🎯 Objective

Generate business insights using sales aggregations.

---

## ✅ ANALYSIS PERFORMED

Calculated overall revenue generated by each product.

### Total Sales by Category

### Total Sales by City
Generated city-level sales insights.

- Category Sales Report
- City Sales Report

# 🔹 REQUIREMENT 2 – CUSTOMER INSIGHTS

## 🎯 Objective

Analyze customer purchasing behavior.

---

## ✅ ANALYSIS PERFORMED

### Number of Orders per Customer
Calculated total orders placed by each customer.

### Top Customers Based on Spending
Identified high-value customers using total purchase amount.

---

## 📊 OUTPUTS GENERATED

- Customer Order Count
- Top Spending Customers

---

# 🔹 REQUIREMENT 3 – DATA QUALITY CHECKS

## 🎯 Objective

Ensure clean and reliable data before analytics.

---

## ✅ DATA QUALITY RULES APPLIED

### NULL Handling
- Replaced NULL amount values with default values
- Handled missing city values using "Unknown"

### Negative Value Removal
- Removed records with negative sales amounts

### Duplicate Removal
- Removed duplicate order records

### Critical Column Validation
Validated important fields such as:
- order_id
- customer_id
- product
- amount

---

## 📊 CLEAN DATA OUTPUT

Generated cleaned DataFrame with:
- No NULL values
- No duplicate records
- No invalid negative sales

---

# 🔹 REQUIREMENT 4 – LATEST DATA HANDLING

## 🎯 Objective

Keep only the latest record when duplicate order IDs exist.

---

## ✅ TECHNIQUE USED

Applied:
- Window Functions
- ROW_NUMBER()

Partitioned by:
- order_id

Ordered by:
- latest date

---

## 📖 BUSINESS LOGIC

If multiple records exist for the same order:
- Keep the most recent transaction
- Remove older versions
